# 一个 3 小时做出来的网站，怎么变成月入 3.5 万美元的目录生意

我叫 Pontis。

我和最好的朋友 Victor 一起做过很多开源项目，也在长期自举创业（**不依赖外部融资，靠收入和现金流把公司做起来**）我们的主项目 Midday。

但这篇文章要讲的不是 Midday，而是一个我们在周末做出来的小项目：Cursor 目录站。

Cursor 目录站是一个目录网站（**把某类资源集中整理、分类、搜索和展示的网站**）。它帮助 Cursor 开发者发现、搜索和使用 Cursor 规则（**写给 Cursor 这类 AI 编程工具的项目约束、代码风格和工作方式说明**）。

这个网站最早只花了 3 小时做出第一版。

现在它每月收入大约 3.4 万到 3.5 万美元，毛利率约 99.8%，上线以来有 2.2 220 万独立访客，注册用户约 4.9 万。运营成本不到 500 美元一个月，维护时间大约每月 3 小时。

这件事最反直觉的地方是：目录网站听起来已经过时了。

但在对的时间、对的人群、对的问题里，目录站仍然可以是一个非常强的产品形态。

## 机会来自一次飞往法国的航班

这个想法出现在我搬去法国的路上。

那是一趟 3 小时航班。飞行途中，我下载了很多技术相关视频来看，其中包括一些 Cursor 相关内容。

看着看着，我发现一个共同模式：很多开发者都在讨论 rules。

这些 rules 对 Cursor 用户很重要。它们可以告诉 AI：这个项目用什么技术栈、代码应该遵循什么风格、哪些目录不能乱改、测试应该怎么跑、组件应该怎么写。

但当时没有一个清晰的地方可以集中找到这些规则。

你可以用 Google 搜到一些 GitHub gist，也可以在论坛里翻到零散帖子，但没有一个真正好用、清楚、可发现的入口。

我一下子觉得：这里应该有一个目录。

飞机落地后，我到了 Airbnb，马上给 Victor 打电话。我说：我们得做一个 Cursor rules 的目录站。

Victor 立刻打开 Figma 开始设计，我用 Next.js 初始化项目，然后先写了一个硬编码 JSON 文件（**不先接数据库，而是把初始内容直接写在代码里的简单数据文件**），里面放了一些我在线上找到的 rules。

半小时左右，我们已经有一个可以点的版本部署到了 Vercel。

随后我买下了 cursor 目录站这个域名。

整个第一版大约 3 小时完成。

## 为什么 3 小时能做出来

这不是因为产品本身神奇，而是因为前置能力已经积累很久。

我和 Victor 合作很多年，已经非常清楚彼此的工作方式。Victor 负责设计，我负责代码。我们也已经有现成的设计系统（**可复用的视觉规范、组件和交互模式**）和组件可以复用。

所以，我们不需要从零讨论按钮长什么样、页面结构怎么排、技术栈怎么选。

我们只需要把机会变成可访问的网站。

这也是很多人误解“快速发布”的地方。

快速不是随便做。

快速来自长期积累后的低摩擦执行。你熟悉自己的技术栈，有可复用组件，知道部署流程，知道页面的基本信息架构，就能把一个很小但完整的版本迅速推出去。

当时我们没有给这个项目设太多期待。

我们只是觉得：这个需求存在，现在没人把它整理好，那就先发出去。

## 它为什么会传播

第一版上线后，我们在 X 上发布。

我们一直公开构建（**把产品进展、代码、想法、数据和过程公开分享**），所以本来就有一些开发者关注我们。Cursor 目录站的第一批传播来自 X。

其中一条帖子拿到了大约 100 万曝光。

接着，我又把它发到 Hacker News。它上了首页。

之后，有很多 YouTuber 开始在视频里介绍这个网站。

这几个渠道叠加后，Cursor 目录站很快进入开发者社区视野。

它传播起来还有一个很重要的原因：产品本身和目标人群天然匹配。

Cursor 用户本来就是开发者，而开发者喜欢收藏工具、分享资源、研究别人怎么设置工作流。一个开源、设计漂亮、能直接复制规则的网站，很容易被他们转发。

更重要的是，Cursor 当时正处在上升期。

当一个大趋势刚起来时，围绕它的基础设施、资源整理、模板、教程、规则库，都有机会被快速传播。

目录网站的核心不是“把东西堆起来”，而是在一个大家同时困惑的节点，帮他们把分散资源整理成可发现、可搜索、可使用的形式。

## 设计成了竞争优势

很多目录网站失败，是因为它们看起来像临时拼出来的列表。

Cursor 目录站不一样。

它不是只有资源堆叠，而是有清楚的界面、明确的分类、搜索、热门规则、官方规则、用户提交和投票。

我越来越相信，在 AI 让建站变得更容易之后，设计反而更重要。

因为现在谁都能很快生成一个网站。结果是，大量 AI 生成的网站看起来非常相似。它们有同样的卡片、同样的渐变、同样的布局、同样的空洞文案。

你不需要做得花哨，但你必须有自己的调性（**产品在视觉、语言、节奏和细节上给人的整体感觉**）。

我们的设计方法并不复杂。

保持简单。找到自己喜欢的网站。学习它们的模式、间距、字体、信息层级和细节。然后把这些东西消化成自己的组件和系统。

如果你刚开始做产品，我建议你先收集一批真正喜欢的网站，不要只看表面效果，而要拆解：为什么这个页面看起来清楚？为什么这个按钮位置合理？为什么这个列表让人愿意继续浏览？

设计不是装饰。

在这种容易被分享的开发者工具里，设计本身就是分发的一部分。

## 目录网站在 2025 年还有效吗

我经常被问：目录站是不是已经死了？

我的答案是：没有。

但前提是你要做对人群、对时机、对资源类型。

目录解决的是一个长期存在的问题：用户想找到某类东西，但这些东西分散在不同地方，质量参差不齐，搜索成本很高。

在 AI 工具爆发后，这个问题反而更明显。

新工具、新协议、新 rules、新 workflows 出现得很快。用户需要一个地方帮他们理解、筛选、对比和使用。

如果你能在一个浪潮刚起来时抓住它，并成为某类资源的默认入口，目录站仍然很强。

Cursor 目录站后来不只是收录 rules。MCP 出现后，我们又加入了 MCP（Model Context Protocol，**让 AI 工具连接外部数据源和工具的协议**）相关的可搜索资源。这个变化让网站又获得了一波采用。

这说明目录网站不能只是静态列表。

它要跟着用户需求和生态变化继续扩展。

## Cursor 目录站怎么赚钱

网站本身有几个核心模块。

用户可以浏览不同 cursor rules，复制后放进自己的编辑器配置里。也可以按兴趣筛选、查看热门内容、查看官方内容、搜索具体规则。

我们开放注册，让用户创建账号，提交自己的内容。其他人可以投票，社区本身开始贡献资源。

之后，我们加入了招聘板块。公司可以付费发布开发者相关职位。

我们还加入了精选展示位（**付费让某个资源在网站上获得更高曝光的位置**）。比如某些 MCP 或开发工具可以付费出现在更显眼的位置。

另外，我们还做了一个生成 cursor rules 的功能。用户可以上传 JavaScript 项目中定义依赖和脚本的配置文件，系统根据项目依赖生成适合这个项目的 rules。

这个功能来自一个很明确的观察：很多开发者知道自己想让 Cursor 更贴合项目，但不知道怎么开始写 rules。

所以，我们不仅提供别人的规则，也帮用户生成自己的规则。

这类变现方式和目录本身是匹配的。

目录有流量，流量来自明确人群，明确人群又吸引招聘方、工具方和生态服务方。只要广告或展示位不破坏用户体验，它就能成为很轻的收入来源。

## 技术栈并不复杂

我们坚持用自己熟悉的东西。

Cursor 目录站用 Next.js 和 TypeScript 构建，UI 组件主要用 shadcn/ui。邮件用 Resend，分析用 OpenPanel，代码托管在 GitHub，部署在 Vercel。

支付方面，我们用 Polar。它是记录商户（**帮你处理支付、税务和部分合规责任的服务商**），对欧洲公司尤其方便。

日常协作上，我们用 Cursor 写代码，Notion 做产品管理，GitHub 做版本控制。

所有工具选择背后都有同一个原则：no servers, 少管服务器（**少制造摩擦**）。

我们不想把时间花在基础设施上。这个项目本来就是一个轻量目录，最重要的是快速发布、快速迭代、跟上生态变化。

总成本大约每月 525 美元。

在每月 3 万多美元收入下，毛利率接近 99.8%。

## 我从这个项目里学到的事

第一，快不是因为草率，而是因为你已经有足够多的可复用能力。

如果你每次都要重新选技术栈、重新想设计、重新研究部署，3 小时很难做出东西。真正的速度来自积累。

第二，目录网站没有死，但泛泛的目录很难活。

“所有 AI 工具大全”这种东西很容易被复制。更好的机会是围绕一个快速增长、用户非常明确、资源高度分散的生态做深。

第三，发布时机很重要。

Cursor 目录站不是在 Cursor 已经完全成熟后才做，而是在大家刚开始研究 rules、资料还很分散的时候出现。它抓住的是一个窗口期。

第四，公开构建可以放大早期项目。

如果我们只是默默上线，可能不会有那么多第一波流量。X、Hacker News、YouTube 覆盖让它迅速进入目标用户视野。

第五，设计在 AI 时代更重要。

当“做出来”变得容易，“看起来可信、好用、值得分享”就会成为更大的差异。

如果今天你也想做一个小网站，不要先问“目录还行不行”。更好的问题是：有没有一个正在变热的领域，用户正在到处找同一类东西，但还没有一个清楚、漂亮、可搜索、可参与的入口？

如果答案是有，那就先做一个小版本发出去。

你永远不知道它会不会失败，也永远不知道它会不会变成月入 3.5 万美元的生意。

但不发布，你就不会知道。
