# 我靠开源，把一个社媒排程工具做到月入 1.7 万美元

我做的产品叫 Postiz，是一个开源的社交媒体排程工具。

简单说，它可以让你把内容一次性安排到 25 个社交媒体平台上，还带了一些 AI 辅助功能。现在，Postiz 每月收入大约 1.7 万美元，有 472 个付费订阅用户。

听起来这像是一个很拥挤的市场。

社媒排程工具并不新鲜。市面上已经有很多成熟产品，几乎每个做内容、做营销的人都听过类似工具。可我仍然能在这个市场里跑出来，一个很重要的原因就是两个字：开源。

很多人理解开源软件时，只会想到“代码公开”“别人可以免费用”。但我真正想讲的是：开源不仅是一种开发方式，也是一种分发方式。

如果做得好，它可以把你的项目介绍给成千上万的开发者，帮你获得反馈、口碑、内容、搜索流量，甚至帮你建立品牌信任。

Postiz 到现在已经被下载超过 500 万次。对一个小团队来说，这种分发能力非常难靠传统广告买到。

## Postiz 到底是什么？

Postiz 是一个 SaaS 产品，同时也是一个开源项目。

SaaS 是 Software as a Service，也就是软件即服务。用户不用自己部署软件，直接付费使用云端版本。

Postiz 的云端版本和开源版本功能基本一样。开发者可以把代码下载到自己的机器上，自行部署、自行配置，然后使用完整产品。但如果他们不想处理这些麻烦，也可以直接购买我们的云服务。

这正是我对开源的理解：它有点像 SaaS 的免费层，但你不用承担对方的使用成本。

如果用户选择自托管，他们需要自己配置服务器、连接各个平台、处理各种授权和部署问题。对很多开发者来说，这没问题；但对普通公司和营销团队来说，付费使用云端服务反而更省事。

我们的收费模式也很传统：标准版、团队版、专业版、旗舰版。不同套餐支持不同数量的社媒渠道和功能。我们希望用户从标准版开始用，再根据更高级的功能升级，而不只是按渠道数量逼他们付更多钱。

目前 Postiz 的关键数据大概是这样：

月经常性收入（MRR，Monthly Recurring Revenue）约 1.7 万美元。

付费订阅用户 472 个。

从 2025 年 8 月开始，累计试用用户 3830 个。

试用到付费的转化率约 21%。

流失率现在还很高，大约 19%，这是我们正在努力降低的问题。

这不是一个完美业务，但它已经证明了一件事：在一个竞争激烈的 SaaS 市场里，开源可以成为非常有效的切入角度。

## 我为什么会走进开源

一开始，我并不是所谓“开源圈的人”。

我只是一个做了很多年开发的程序员。后来，我在一个联合办公空间里认识了一些很有意思的人，他们正在做一个开源的基础设施工具，用来发送通知。

我加入了那家公司，在那里待了两年。那两年里，我们把项目从 0 做到 GitHub 上 3.2 万颗星。

那是我第一次真正看到开源的增长速度。

GitHub stars 可以理解为开发者对项目的公开收藏和认可。它不等于收入，但它代表关注度、信任度和传播势能。

从那以后，我看到越来越多公司开始走开源路线。第二年，我开始做开源相关咨询。因为当时专门讲“开源怎么增长、怎么营销”的人并不多，所以我做了很多非常垂直的咨询。

但咨询做久了以后，我压力很大。每天开会、聊天、解释策略，反而让我远离了自己最喜欢的事情：做产品。

于是我决定回到产品本身，开始做 Postiz。

既然我已经亲眼见过开源的增长能力，也帮很多公司做过开源咨询，那我自己的产品当然也要开源。

## 为什么 2026 年更适合做开源

我认为，现在比过去任何时候都更适合做开源。

原因很简单：今天，几乎每个人都能做软件了。

AI 编程工具降低了构建门槛。你能做的东西，别人也能做。一个产品如果只靠代码本身，很难形成长期壁垒。

这意味着，市场会变得越来越拥挤。

每天都有新的 SaaS 出现，每个品类都有一堆替代品。如果你只是再做一个普通工具，用户很难记住你，也很难相信你。

开源给了你一个差异化入口。

它能把项目带到开发者面前，而开发者虽然通常不是你的直接付费用户，但他们会帮你做几件非常重要的事。

第一，他们会帮你建立品牌。

开发者看到一个项目是开源的，会更愿意研究、讨论、分享。他们可能不会付钱，但他们可能在公司里把你的工具推荐给真正需要的人。

第二，他们会带来口碑内容。

我每天在社交平台搜索 Postiz，经常能看到开发者发帖介绍它、评测它，甚至帮我“销售”它。

第三，他们会贡献反馈。

我不能说外部贡献者一定会让你开发效率暴涨。很多时候，你仍然要花时间审核、沟通和修正。但他们会帮你更快发现 bug，提出功能建议，也会让产品社区更活跃。

第四，他们会带来 SEO。

SEO 是 Search Engine Optimization，也就是搜索引擎优化。很多人会写“如何使用 Postiz”“Postiz 评测”“某某开源社媒工具对比”之类的文章。这些内容会长期在搜索引擎里给你带来流量。

第五，开源会增强非开发者的信任。

很多普通用户看到 GitHub 上最近还有提交、有人提 issue、有人贡献代码，会觉得这个产品是活的、可信的。

第六，开源还可能带来企业收入。

很多企业不愿意把数据放在外部云服务里，尤其是涉及安全、合规、客户资料的场景。他们会希望自托管，但又需要官方支持。这就是开源企业支持的机会。

对一些开源公司来说，最大的钱往往就在这里。

## 不要太怕别人复制你

很多人不敢开源，是因为怕别人复制。

我理解这种担心。Postiz 也被复制过。有人直接拿我们的代码，自托管成一个看起来完全不同的解决方案，然后试图和我们竞争。

刚开始我也会紧张。

但我观察一两周后发现，很多复制者很快就放弃了。

因为代码不是全部。

真正有价值的是品牌、持续更新、社区信任、用户反馈、文档、支持、分发能力，以及你对这个产品长期投入的决心。

别人可以复制你今天的代码，但他永远落后你一步。他没有你的社区，没有你的品牌，也没有你对问题的理解。

在 AI 能写代码的时代，代码的稀缺性会下降。品牌和分发的重要性会更高。

开源恰好是一种低成本建立品牌和分发的方式。对自举创业者尤其有用。自举创业者指的是不靠大量外部融资，主要靠自己收入和现金流推进业务的人。

## 我的开源启动方法论

如果今天让我从头再做一个开源项目，我会按一套很明确的步骤来。

第一步，把 GitHub 当成你的营销落地页。

很多人只把 GitHub 仓库当代码仓库，这是错的。

对开源产品来说，GitHub 就是你的第一张门面。README 要写得像官网一样清楚，甚至比官网更重要。

你要让人一眼看懂：你做了什么，解决什么问题，适合谁用，怎么开始。

如果你做的是一个已有产品的替代品，最直接的表达方式是：我是某某产品的开源替代方案。比如“某某工具的开源替代品”。这会让用户立刻有上下文。

第二步，选好许可证。

常见的开源许可证包括 MIT、Apache 2.0 和 AGPLv3。它们对使用、修改、商业化和代码开放义务的要求不同，不能随便选。

许可证本质上是你允许别人如何使用代码的规则。选错了，后面商业化可能会很麻烦。

第三步，提前创建一些 issue。

很多开发者来到你的仓库，并不知道应该贡献什么。如果你提前列出一些具体功能、bug 或任务，他们就更容易参与。

不要期待别人自己读完整个代码库，然后主动发现该做什么。你要给他们一个入口。

第四步，开一个 Discord 社区。

开发者需要快速进入一个地方提问、交流、反馈。Discord 很适合做这种实时社区。你可以参考成熟开源项目的频道结构，不需要从零发明。

第五步，写好部署文档。

如果开发者不知道怎么部署，他们很快就会离开。

开源项目最怕“看起来不错，但跑不起来”。最好提供 Docker 部署方式。Docker 可以把应用需要的环境打包起来，让部署变得更简单。（**可以理解为把软件和运行环境一起装进一个标准容器，减少“我电脑上能跑，你电脑上跑不了”的问题。**）

这些准备完成后，才进入真正的发布阶段。

## 发布的目标：短时间集中流量

开源发布不是随便发几条帖子。

你的目标是：在最短时间里，把尽可能多的流量集中到 GitHub 仓库。

因为这样你才有机会进入 GitHub Trending。GitHub Trending 是 GitHub 的热门项目榜单，一旦进去，星标数、访问量、Discord 用户、issue、pull request 和 fork 都会快速增加。

为了做到这一点，我会在发布前准备几个渠道。

首先，提前两周注册 Hacker News 账号。

Hacker News 是开发者和创业者很集中的社区。发布时，用“Show HN: 项目名”的格式提交，并且链接到 GitHub 仓库，而不是官网。Hacker News 对内容很挑剔，但它很喜欢有价值的开源项目。如果成功进入首页，可能带来接近 1 万次访问。

其次，提前准备 Reddit 账号。

Reddit 对新账号和低 karma 账号不友好。如果你没有 karma，帖子很容易被过滤。karma 可以理解为 Reddit 里的社区信誉分。

对开源项目来说，r/selfhosted 是非常好的社区。很多地方禁止自我推广，但 r/selfhosted 的用户本来就在等新项目、新版本和可自托管工具。

你可以每次发布新版本时再发一次，说明新增功能，并且真诚地请大家给 GitHub star。

这里的语气很重要。不要装成大公司。用第一人称，像公开构建一样讲：我做了什么，为什么做，想听大家反馈。

如果项目和 AI 相关，还可以考虑 r/LocalLLaMA、r/AI_Agents、r/LangChain 等社区。

第三，使用 Lemmy。

Lemmy 是一个类似 Reddit 的开源、去中心化社区平台。它在很多方面未必比 Reddit 强，但对开源项目营销非常友好。我每次在 Lemmy 发开源相关内容，通常都能拿到至少 100 个 upvote。

第四，写技术文章。

我会在 dev.to、Medium、Hackernoon 这些平台写文章。内容可以是“我是如何构建某个功能的”，也可以是“我刚发布了一个开源项目，解决某个问题”。

后续还可以写榜单类文章，比如“2026 年值得关注的 10 个开源项目”。这类文章不一定非常精准，但能带来 star 和曝光。

这里有一个容易被忽略的流量来源：Google Discover。

Google Discover 会根据用户兴趣推荐文章。我的很多 dev.to、Medium、Hackernoon 文章，主要流量都来自 Discover。所以标题和封面图很重要，就像做 YouTube 缩略图一样重要。

第五，把你已有的渠道都导向 GitHub。

如果你有 X、LinkedIn、邮件列表，就在同一周集中发布，把流量都送到 GitHub 仓库。

重点是“同一周”。

分散发布会让每个渠道的效果被稀释。集中发布才更容易冲上热门榜。

## 产品本身也要能打

开源能带来分发，但产品不能只靠概念。

Postiz 的核心功能其实很直接：连接多个社交媒体账号，创建内容，选择平台，安排发布时间。

如果某个平台支持评论或不同格式，我们也会让用户分别编辑。比如 Instagram 可以发帖子或 Reels，不同平台的内容展示方式不同，所以用户可以针对单个平台微调。

我们还做了日历视图，让所有草稿和排程内容都能被拖动、调整日期。

在基础功能之外，我们加入了一些 AI 工具，比如生成图片、生成视频、设计媒体、优化文本等。

另一个重要功能是公共 API。

很多开发者愿意用 Postiz，是因为它开源，而且可以通过 API 自动化发布流程。比如他们可以把 Postiz 接到 n8n 工作流里，也可以自己写脚本调用 API。

n8n 是一个自动化工作流工具，可以把不同服务串起来。（**类似“如果 A 发生，就自动执行 B 和 C”的流程编排工具。**）

## 我的技术和运营栈

Postiz 的后端和主要服务托管在 Railway 上。营销网站使用 Vercel，只跑 Next.js 应用，不使用 Vercel API，因为我们有自己的后端。

SEO 方面，我用 Semrush，也用 Ahrefs 的部分仪表盘功能，每月大约 20 美元。

邮件发送使用 Resend。

文件托管用 Cloudflare R2。

短链接和联盟计划用 dub.co。

数据分析用 Plausible。

AI 图片和文本生成用 OpenAI、Kie.ai 和 Fal.ai。

最大的一项成本来自视频转码。因为不同社交媒体平台对视频尺寸和格式要求不同，如果不转换到正确规格，发布就会失败。为了保证帖子能顺利发出去，我每月大约花 600 美元在这部分。

我们还用 Outrank 做 SEO，每天产出一篇 SEO 文章。

开发工具方面，我用 Cursor 做 AI 辅助编程，但不是重度用户，每月 20 美元。IDE 用 WebStorm。邮件 newsletter 用 Beehiiv。

客服主要在 Discord，成本是 0。

因为我们是开源项目，GitHub Copilot 对开源免费，Sentry 的错误监控也可以免费使用。GitHub Actions 用来跑测试和构建 Docker 版本，也能借助开源额度。

整体利润率大约 80%。

## 我最后想给的建议

如果回到最开始，我会告诉自己：先学习，再构建。

这句话听起来有点有争议。因为很多创业建议都会说“不要想太多，赶紧做”。

我同意不能陷入分析瘫痪。分析瘫痪指的是一直研究、比较、计划，却迟迟不动手。

但完全不学习也很危险。

我早期做过很多失败的项目。后来我停下来，开始认真读营销、增长、销售和商业模式相关的书。比如关于 traction、关于高价值 offer、关于线索生成的内容。

学完这些基础知识后，我做决策明显更聪明。

很多人一上来就做 B2C，因为看起来用户多、市场大。但如果你不擅长分发，B2C 往往更难。很多人苦干一年，最后才发现自己应该做 B2B。

所以我的建议是：一半时间学习，一半时间构建。

尤其在开源领域，你不能只会写代码。你还要理解分发、社区、文档、品牌、许可证、定价和支持。

开源不是把代码扔到 GitHub 上，然后等用户自动出现。

真正有效的开源，是把 GitHub 当落地页，把社区当分发渠道，把透明度当信任资产，把品牌当长期壁垒。

这就是我用 Postiz 学到的最重要一课。
