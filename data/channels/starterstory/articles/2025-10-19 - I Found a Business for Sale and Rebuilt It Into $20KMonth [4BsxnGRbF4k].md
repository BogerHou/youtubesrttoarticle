# 我看到一个待售生意，没买它，而是重建成月入 2 万美元的 SaaS

很多人问我最多的问题是：怎样才能想到一个好商业点子？

我的答案是：你不一定需要自己想。

我叫 Adrian，是一个来自 Austin 的独立开发者。我做了一个 SaaS，现在每月收入大约 2 万美元。

这个点子不是我凭空发明的。

我是在一个线上生意交易平台上看到一个成功 App 正在出售。它的数据很好，但我没有买下来，而是自己重新做了一个类似产品。

一年后，这个产品每月收入大约 2 万美元。

我的观点可能有点反直觉：如果一个东西已经有效，你有义务去复制它。

当然，这里的复制不是逐字照抄，也不是把别人的网站、文案、代码原样搬走。真正要复制的是概念、商业模型和已被验证的需求，然后用自己的方式执行得更好。

## 我的背景：从工程师到独立创业

我搬去旧金山，是因为想进入科技行业。

后来我在 App Academy 学写代码。那是一个编程训练营。

毕业后，我在 Utah 做了 3 年工程师。但我一直有一个目标：最终做自己的生意。

后来我带着 3 万美元存款辞职，开始自由职业，也做过课程和一些产品。

但真正成功，是在我坚持做 Scrape Creators 之后。

这是一个网页抓取 API。它主要抓取社交媒体公开数据，比如 Instagram、YouTube、Twitter，以及一些广告库数据，比如 Facebook Ad Library、LinkedIn Ad Library 等。

现在它每月收入大约 2 万美元。

它不是订阅制，而是点数付费模式。用户先买点数，然后按请求消耗。

目前有大约 600 个付费用户，但他们不一定每月都持续付费。产品每月处理接近 2000 万次 API 请求。

## 我是怎么发现这个点子的

有一天，我 Twitter 上的一个关注者私信我，让我去看 MicroAcquire（现在也常叫 Acquire.com）上的一个待售项目。

那是一个数据抓取 API。

他之所以发给我，是因为我之前已经做过和社交媒体相关的产品，所以他觉得我可能会感兴趣。

我点进去一看，数据让我很震惊。

我以前并不确定，一个 scraping API 能赚这么多钱。

但这个待售产品每月经常性收入大约 3 万美元，已经运营 3 年，而且客户主要来自 SEO（Search Engine Optimization，搜索引擎优化）。

这几个信号放在一起，我立刻觉得：这就是我要做的东西。

原因很简单。

第一，我懂这个领域。

过去 3 年，我一直在研究数据抓取，也已经做过类似 API。技术上，我相信自己能做出来。

第二，它已经被市场验证。

一个产品能做到 3 万美元 MRR（Monthly Recurring Revenue，月经常性收入），并且持续 3 年，不是偶然。

第三，它客户数不到 100 个。

这说明它不需要非常庞大的用户规模，也能产生不错收入。对一个独立开发者来说，这很重要。

第四，它主要靠 SEO 获取客户。

这意味着需求是主动搜索出来的，而不是完全靠广告砸出来的。我当时还有一点 Twitter 影响力，所以我想，即使 SEO 没那么快起效，我也可以直接在 Twitter 上找用户。

## 我怎么快速做出第一版

我是 Node.js 开发者，主要写 JavaScript。

所以第一版其实很直接：把已有的抓取脚本部署成 API。

我把这些抓取逻辑放到 Node.Node.js 服务器上，托管在 Render.com。

文档部分，我没有做复杂系统，而是直接放在 Notion 文档里。

网站也很基础。

整个产品很非常简陋（**但核心可用**），大概几周就做出来了。

几周后，我拿到了第一个客户。

这也是我想强调的一点：如果你已经验证了需求，不要一开始就把东西做得很复杂。

先把核心能力交付出来，让用户能付钱使用。

## 我是怎么找到客户的

很多开发者能把产品做出来，但卡在获客。

我的增长一开始主要来自 Twitter。

我经常在那里发内容，别人会看到我。

第一个客户其实是意外来的。我抓取了一家公司的站点，发了一条相关内容，然后那家公司的 CTO 在帖子下评论。这个互动最后变成了客户。

后来，只要我看到有人发布和社交媒体数据抓取相关的视频或产品，我就会去评论：

我可以给你 1 万免费点数，你可以试试我的 API。

scraping API 有一个很好的特点：你不需要很多客户，也能做到不错的收入。

我现在大部分 MRR（Monthly Recurring Revenue，月经常性收入）其实来自大约 12 个主要客户。

这和很多消费者 App 不一样。消费者 App 往往需要大量用户，而 API 工具只要抓住高价值客户，就能形成可观收入。

## 如果今天重新开始，我会怎么找点子

如果我今天重新开始，我会用一套很具体的方法。

第一步，打开 MicroAcquire 线上生意交易市场。

第二步，筛选 SaaS。

第三步，按要价筛选。

我不会看那些完全不赚钱的项目。我要找的是已经被市场验证的点子。

比如你可以把要价筛到至少 30 万美元以上，或者按 ARR（Annual Recurring Revenue，年度经常性收入）筛选。

第四步，找你擅长的市场。

我当时知道自己想做 web scraping，因为这是我的细分领域。你也应该找自己有知识、经验或兴趣的领域。

第五步，反向找到真实网站。

MicroAcquire 上通常不会直接显示网站名字，但很多 SaaS 很容易被找到。

你可以复制项目信息里的标题、描述、H1 或关键词去 Google 搜索。很多卖家会把网站上的文案直接放到 listing 里。

还有一种办法，是看 listing 里有没有提到竞争对手。

如果有，就去 Google 搜“竞争对手名 + alternative”或“竞争对手名 + alternatives”。很多 SaaS 会做“某某竞品替代品”页面，你就能反推出它的网站。

找到网站后，再对照文案和描述，基本就能确认是不是同一个产品。

第六步，反向分析它怎么获客。

这一步最重要。

产品本身通常不是最难的，难的是客户从哪里来。

你要尽可能读它网站上的所有信息，看它有没有透露获客渠道。然后去找创始人的 Twitter、LinkedIn、播客、YouTube 采访，看他有没有讲过增长方式。

如果你发现它主要靠 SEO，那么你就要判断自己能不能也做 SEO，或者有没有其他渠道能先拿到客户。

第七步，真的把产品做出来。

第八步，不要逐字照抄。

不要复制别人的文案、设计、代码。你要复制的是概念和需求，不是表层材料。

第九步，不要分心。

每天做一件事来推广或改进产品。

你已经知道这个点子能赚钱，接下来拼的就是执行。

## Scrape Creators 到底做什么

Scrape Creators 抓取的是社交媒体公开数据。

比如 Instagram、YouTube、Twitter，以及 Facebook Ad Library、LinkedIn Ad Library 这样的广告库。

这类工具主要服务开发者，因为 scraping 很麻烦。

社交媒体网站经常变化，抓取逻辑容易坏。你需要处理基础设施、代理轮换（**用不同代理 IP 发送请求以提升稳定性和避免限制**）等问题。

我们做的事情很简单：我们来抓，你不用自己抓。

客户包括个人主页链接工具、短视频数据分析工具、社交媒体分析产品等。

收费方式是按点数付费。

目前有几个套餐：

10 美元买 5000 credits。

50 美元买 25000 credits。

500 美元买 500000 credits。

这个产品能做起来，我认为有两个原因。

第一，它确实好用。

市面上很多社交媒体抓取工具经常坏。我的产品比较稳定。

第二，我沟通很及时。

如果服务不稳定，我会频繁和用户沟通。用户也很容易找到我。

很多开发者做这类产品，连邮箱或联系方式都很难找。对 B2B 工具来说，能联系到真人本身就是优势。

## 技术栈和成本

技术栈并不复杂。

后端主要是 Node.js 和 JavaScript，本质上是一堆 HTTP HTTP 请求。

我用一个叫 impit 的包，它由 Apify 开发，用来处理 HTTP 请求。

代理方面，我主要用 4 个服务：Evomi、Core Residential、Webshare 和 Massive。

托管方面，大部分服务在 Render.com，有些子脚本跑在 AWS Lambda。

我也用 Cursor，每月大约 20 美元。数据库用 Supabase。前端是 Astro 加 React。

利润率大约 80%。

主要成本是代理。现在每月代理成本大约 1500 美元。

我还雇了一名菲律宾开发者，晚上帮我监控 API 是否宕机，每月大约 500 美元。

服务器成本大约 400 美元。

对一个每月收入 2 万美元左右的 SaaS 来说，这样的成本结构是健康的。

## 我最大的建议：别再跳来跳去

如果你在 2025 年从零开始，我最想说的是：停止在各种点子之间来回跳。

选一个东西，每天做。

每天专注它。

我自己过去也犯过这个错误，总是被新点子吸引，做一阵就换方向。

但真正让我成功的，是我终于坚持在一个具体方向上持续执行。

很多人还在等所谓“天才点子”。

我的经历说明，创业神话可以反过来理解：你不一定要发明一个全新的东西。你可以先找到一个已经赚钱的模型，理解它为什么有效，然后做出自己的版本，并且执行得更好。

点子不是最稀缺的。

持续执行，才是。
